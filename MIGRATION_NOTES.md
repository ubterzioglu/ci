# Wix to Next.js Migration Notes

## Overview

Çi Neo Cucina's website was migrated from Wix to Next.js 16 + Supabase. This document captures what was exported, decisions made, and what remains incomplete.

## Source Exports

The legacy Wix website data was exported to the `/ref` directory:

- **Pages markdown** (`ref/content/pages/`): Converted HTML/text from public pages
- **Menu data** (`ref/content/menu-items.csv`): Menu items with categories
- **Navigation** (`ref/content/navigation.json`): Site navigation structure
- **Contact info** (`ref/content/contact.json`): Phone, email, region
- **SEO metadata** (`ref/content/seo.json`): Page titles and descriptions for each route
- **Image assets** (`ref/content/image-assets.json`): Manifest of Wix CDN image URLs
- **Audit notes** (`ref/audit/fix-list.md`): Issues identified in the source

## Migration Decisions & Assumptions

### 1. Turkish-First Language

**Decision**: Site defaults to Turkish (TR).

**Rationale**: The source Wix website was entirely in Turkish; the restaurant operates in Turkey. English and German translations are planned via DeepL but are optional expansions.

**Assumption**: Chef Simge Manacıoğlu and the restaurant team are comfortable with Turkish-first branding.

### 2. Currency: Turkish Lira (TRY ₺)

**Decision**: Menu prices are stored in TRY; no EUR or other currencies are supported initially.

**Rationale**: The restaurant is located in Kaş, Antalya, Turkey. All exported menu prices were in TRY.

**Assumption**: No multi-currency pricing is required at launch.

### 3. About Page Slug: `/about`

**Old Wix slug**: `/about-1`

**New slug**: `/about`

**Rationale**: Cleaner, more SEO-friendly slug for the new site.

**Implementation**: A 301 redirect from `/about-1` → `/about` is configured in the `redirects` table to preserve any existing search rankings or backlinks to the old URL.

### 4. Separate Contact Page

**Decision**: Created a dedicated `/contact` page with a contact form.

**Rationale**: The source Wix site did not have a separate contact page; contact details appeared only in the footer. A standalone contact page improves UX and SEO.

**Form fields**: name, email, phone, subject, message (validated, submitted to `contact_messages` table).

### 5. Footer Attribution Removed

**Wix footer text**: © 2035 by Michael Bolano. Powered and secured by Wix

**Decision**: Removed this footer text entirely.

**Rationale**: The Wix attribution is not relevant to the new site and was likely auto-generated by Wix.

**New footer**: Custom footer with restaurant branding, navigation, social links, and legal links to `/impressum` and `/datenschutz` pages.

### 6. Allergen Tags: Inferred

**Decision**: Menu item allergen tags were inferred from ingredient lists in the source export.

**Common inferred allergens**:

- "Contains nuts" or similar phrasing → `nuts`
- "Contains dairy" or "cheese" → `dairy`
- "Contains gluten" or "bread" → `gluten`
- "Contains fish" or seafood items → `fish`
- "Contains shellfish" → `shellfish`

**CRITICAL**: These are inferred from available source data and have **NOT been confirmed by the restaurant**. The restaurant must review and correct all allergen tags before the site goes live.

**Confirmation required**: See `TODO_PANEL_EXPORTS.md` item: "Confirm inferred allergen tags."

## Image Assets: Legacy Wix CDN

**Status**: Wix CDN image URLs are temporarily allowed in `next.config.ts` for backward compatibility.

**Localization process**:

1. All Wix image URLs are catalogued in `ref/content/image-assets.json`
2. Run `pnpm assets:download` to:
   - Download images from Wix CDN
   - Store them locally under `public/images/imported/`
   - Update references in the database (media_assets table)
3. Remove the Wix CDN domain from `next.config.ts` once all images are localized

**Recommended**: Complete image localization before going live (see `TODO_PANEL_EXPORTS.md`).

## Database Schema

The new site uses a Supabase PostgreSQL database with 8 tables:

1. **site_settings** — Key/value JSON config (navigation, hours, social, contact)
2. **pages** — CMS-style page content (markdown + JSON, with SEO)
3. **menu_categories** — Menu categories (e.g., "Topraktan", "Denizden", "Otlaktan")
4. **menu_items** — Menu items with prices, allergens, dietary flags
5. **media_assets** — Image manifest (source Wix CDN URL → local path)
6. **reservation_requests** — Public reservation form submissions
7. **contact_messages** — Public contact form submissions
8. **redirects** — Legacy Wix slug → new slug mappings (301 redirects)

All tables have Row-Level Security (RLS) enabled:

- **Public read** (anon/authenticated): Published pages, active menu, public settings
- **Public insert** only: Reservation requests and contact messages (forms)
- **No public update/delete**: Content managed server-side with service_role key or Supabase dashboard

See `supabase/README.md` for detailed security model and seeding instructions.
